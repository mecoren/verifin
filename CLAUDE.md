# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

Veri Fin (包名 `verifin`) 是一个完全免费、数据自主、本地优先的 Flutter 记账应用。仓库规则详见 `AGENTS.md`（中文），本文件是其摘要加架构说明。**与用户沟通时使用中文。**

## 常用命令

```bash
flutter pub get                        # 安装依赖
flutter run -d <android-device-id>     # 本地预览（Android 模拟器或真机）
flutter analyze                        # 静态检查（flutter_lints）
flutter test                           # 运行全部测试
flutter test test/entries_test.dart    # 运行单个测试文件
flutter test --plain-name "关键字"      # 按名称运行单个测试
dart format .                          # 提交前格式化
```

本地预览与验收只能在 Android 模拟器或真机上进行；正式验收使用 GitHub CI 构建的 release APK（下载到手机安装测试）。

## 发布与 CI

- **不要在本机构建 APK 作为交付物**。Android 安装包只由 GitHub Actions 生成。
- CI（`.github/workflows/flutter.yml`）只在推送 `vX.Y.Z` 标签时触发：analyze → test → build apk --release → 发布 GitHub **预发布**（`prerelease`、不标记 Latest）。普通 `main` 推送不触发。**真机验收通过后，在 GitHub 手动把该 release 改为正式版（取消预发布、设为 Latest）**——CI 默认只出预发布，不直接出正式版。
- 发版用 `scripts/publish.sh patch`（也支持 `minor`、`major` 或显式版本号）；脚本会更新 `pubspec.yaml` 和 `lib/app/app_version.dart` 里的 `appVersionLabel`、提交、打标签并推送。**Windows 上用等价的 `scripts/publish.ps1 patch`（PowerShell）**，两脚本逻辑一致；改动发布流程时须同步维护这两个。
- **发版必须经用户明确同意**：只有用户明说「发布 patch/minor/major 版本」之类时才执行发版流程，绝不擅自发版；打标签会触发 CI 构建，不可逆。
- 发版前先把 `CHANGELOG.md` 顶部的 `## [Unreleased]` 段落改名为本次版本号并加日期（如 `## [1.8.2] - 2026-07-08`），再在其上新开一个空的 `## [Unreleased]`，随发版提交一起提交，然后再跑 `scripts/publish.sh`。
- Release APK 必须用项目内稳定 keystore（`android/app/verifin-release.jks`）签名。

## 提交规范

- 提交信息用 `type: summary` 格式（如 `feat: add login screen`、`fix: correct counter test`）。
- **提交中不要包含任何 AI 署名**（`Generated by ...`、`Co-authored-by: ...` 等）——这是仓库明确规则，覆盖默认行为。
- 每完成一个相对独立的模块就提交一次，不要把所有改动混到最后一次提交。

## 架构

### 状态管理：单一 Controller + InheritedNotifier

全部应用状态集中在 [lib/app/veri_fin_controller.dart](lib/app/veri_fin_controller.dart) 的 `VeriFinController`（`ChangeNotifier`）中：交易、账户、分组、账本、分类、预算、个人信息、主题和各种视图偏好。它通过 [lib/app/veri_fin_scope.dart](lib/app/veri_fin_scope.dart) 的 `VeriFinScope`（`InheritedNotifier`）注入组件树。没有 Provider/Riverpod/Bloc 等第三方状态库。

- 持久化：账目类核心数据（交易、账户、分组、账本、分类、标签、月度/分类/按日预算）**只**存 SQLite，由 [lib/data/](lib/data/) 的 `AppDatabase`（建表 + 版本迁移）与 `LedgerRepository`（接口，整表覆盖式读写）承载，生产实现 `SqliteLedgerRepository`。controller 内存列表仍是读取源，写入经 `_persistX` 落库。`VeriFinController.create()`（唯一构造入口，异步）从库载入；全新数据库首启动播种默认账本/账户/分组/分类（标签默认为空）。交易与标签为多对多，交易表 `tag_ids` 列存 JSON 数组，不建关联表；标签全局共享不分账本。图片附件存独立 `attachments` 表（按 `entry_id` 关联，`data_url` 存压缩 JPEG），不放 entries 表以避免整表覆盖写入放大；删除交易/账户/账本时级联清理附件。没有 KV↔SQLite 迁移，也没有 KV 回退——账目类只认库。偏好类小数据（主题、触感、面板配置、资产排序/折叠、封面、活动账本、个人资料、金额小数位等）仍以 JSON 存入 `LocalKeyValueStore`，键形如 `verifin.theme.v1`。金额是否强制展示两位小数是全局显示偏好（`verifin.amount_format.v1`）：因 `formatAmount` 等格式化入口是无 context 纯函数（小组件/通知/`series_math` 也调用），由 controller 单向同步到 [lib/app/amount_format.dart](lib/app/amount_format.dart) 的顶层量 `amountForceTwoDecimals`，格式化函数读它，零调用点改动即全局生效。修改 `lib/data/` 表结构须提升 `AppDatabase.schemaVersion` 并在 `_onUpgrade` 写迁移。测试注入内存实现 `InMemoryLedgerRepository`（见 `test/support/`）。分类有参照完整性保护（防「幽灵同名分类」，详见 `docs/dev/tech-decisions.md` 的「分类参照完整性」）：`categories` 表有 `(label,type,IFNULL(parent_id,''))` 唯一索引；`VeriFinController.create()` 载入与 `importFromJson` 覆盖后都跑 `_healCategoryData()` 自愈（孤儿 parentId 重挂顶级、重复同名分类合并并迁移引用、悬空交易/规则引用归入「未分类」）；创建分类（`addCategory`、导入 `resolveCategory`）按 `normalizedCategoryLabel` 归一化查重复用——新增分类创建入口须走此查重、勿绕过唯一约束。
- 多账本隔离：交易、账户、分组都带 `bookId`，controller 的 getter 按 `_activeBookId` 过滤后返回不可变列表；月份/分类预算的键带 `bookId:` 前缀（旧数据加载时自动迁移到默认账本）；按日预算是账本级的每日花销上限，键就是 `bookId`（`daily_budgets` 表），当日已花按 `dayExpenseTotal` 聚合。
- 账户类型（`AccountType`）六种，两项能力各由一个 getter 判定、**勿硬编码具体类型**：`supportsCardLast4`（完整卡号 `Account.cardNumber` + 后四位 `cardLast4`）仅信用卡/储蓄卡；`supportsCredit`（信用额度 `creditLimit`、账单日 `statementDay`、还款日 `dueDay`、还款）仅信用卡/信用账户（`creditAccount`，花呗/白条等无实体卡号的信用账户）。「后四位跟随完整卡号」开关是**持久化字段** `cardLast4Follows`（落库 + 进备份，忠实还原用户选择、不靠反推），由受控组件 `CardNumberFields`（`follows`/`onFollowsChanged`）驱动、打开时后四位自动取末四位（`cardLast4Of`）；列表只显示后四位、详情页展示完整卡号可复制。信用额度/本期账单纯函数在 [lib/app/credit_card.dart](lib/app/credit_card.dart)（`usedCredit`/`availableCredit`/`currentBillingCycle`/`billingCycleExpense`）；还款（`CreditRepaymentPage`）本质是「扣款账户→本账户」的转账（`categoryId` 空、可空账户代还）。切换类型丢失对应能力时清空相关字段。完整能力矩阵与取舍见 `docs/dev/tech-decisions.md`「账户类型能力矩阵」。
- 转账（`EntryType.transfer`）同时影响转出/转入账户余额，但不计入收支统计（聚合逻辑在 [lib/app/ledger_math.dart](lib/app/ledger_math.dart)）。转账可带手续费 `LedgerEntry.fee`，由转出账户承担（转出扣「金额+手续费」、转入只加金额）。支出可标记待报销（`reimbursable`）并被退款/报销回款冲抵（`refundedAmount`）：余额与所有统计都按 `LedgerEntry.netAmount`（金额−已冲抵）计算，回款回到原账户、不产生独立收入条目。交易的 `accountId` 可为空串表示「无账户」（只记金额、不计入任何账户余额但仍计入收支统计；转账 `toAccountId` 也可空，如单边转账）——`ledger_math` 按 id 匹配天然兼容，展示层须用 `accountDisplayName`（`demo_data.dart`）而非 `accountById`，避免空 id 回退首个账户误显示；记账 UI（`entry_detail_page.dart` 新建、`transactions_pages.dart` 编辑）与账户选择弹窗（`sheets.dart` 的 `showAccountPickerSheet(noneLabel:)`）均提供「无账户」选项。

### 平台适配：条件导入两件套

Android/测试差异统一用条件导出模式（`stub` + `if (dart.library.io)`）：

- [lib/local_storage/](lib/local_storage/)：`LocalKeyValueStore`（偏好类小数据）— io 用 `SharedPreferences`，测试用内存 stub。真实平台启动必须 `LocalKeyValueStore.create()`，**真实平台绝不能落到内存实现**。
- [lib/data/database_factory.dart](lib/data/database_factory.dart)：SQLite `DatabaseFactory` 与路径解析 — io 用 `sqflite` 原生、stub 抛错（测试注入 `databaseFactoryFfi`）。
- `lib/app/avatar_picker_*.dart`：本地图片选择。
- `lib/app/data_file_port_*.dart`：备份导出/导入的文件读写（Android 默认写系统 Downloads）— io 支持文本写入（`downloadTextFile`，CSV 模板等）与字节写入（`downloadBytesFile`，zip 备份），导入用 `pickBackupBytes` 按字节读入（`.json`/`.zip` 均可）。
- `lib/app/attachment_picker_*.dart`：交易图片附件的拍照/相册选择 — io 用 `image_picker` 拍照或选图并压缩为 JPEG data URL（`image` 包在后台 isolate 缩放），stub（测试宿主）不支持（`attachmentPickingSupported=false`）；UI 组件 `lib/pages/attachments_editor.dart`。
- `lib/app/biometric_auth_*.dart`：应用锁的生物解锁 — io 用 `local_auth`（仅 Android/iOS 生效，不保存生物特征数据；`local_auth` 无法严格排除人脸，故用户文案统一为「生物解锁」），stub（测试宿主）一律不可用。Android 端依赖 `FlutterFragmentActivity`、`USE_BIOMETRIC` 权限与 `minSdk≥23`。
- `lib/app/backup/backup_storage_*.dart`：备份目录选择与文件读写 — io 上 Android 走 SAF（经 `platform_bridge`）、桌面走 `file_selector`+`dart:io`，stub 不支持持久目录。
- `lib/app/backup/webdav_client_*.dart`：WebDAV 备份 — io 用 `dart:io HttpClient` 手写 PUT/GET/PROPFIND/MKCOL，stub 不支持。
- `lib/app/reminder/notification_scheduler_*.dart`：记账提醒本地通知 — io 用 `flutter_local_notifications`+`timezone`+`flutter_timezone`，每日在设定时刻发一条提醒（inexact 调度免精确闹钟权限、`matchDateTimeComponents: time` 每日重复），stub（测试宿主）不支持。配置 `ReminderSettings`（`reminder/reminder_settings.dart`）存 KV（`verifin.reminder.v1`），是设备本地偏好、不进 JSON 备份；`main.dart` 开屏与配置变化时 `apply` 重排通知。Android 端需 `POST_NOTIFICATIONS`/`RECEIVE_BOOT_COMPLETED` 权限、插件接收器与 core library desugaring。
- `lib/app/ai/ai_client_*.dart`：AI 对话记账的聊天补全请求 — io 用 `dart:io HttpClient` POST `{baseUrl}/chat/completions`（OpenAI 兼容、`Authorization: Bearer`、带超时），stub（测试宿主）抛错。配套 `ai_settings.dart`（`AiSettings` 值类：baseUrl/apiKey/model，`chatCompletionsUrl` 拼接容错）与纯函数 `ai_entry_parser.dart`（`buildAiEntryPrompt` 组提示词，`parseAiEntryDraft` 从模型文本提取 JSON 并把分类/账户 id 校验到当前账本清单，未命中降级并回 `AiDraftWarning`，产出 `AiEntryDraft`；提示词带当前日期与时间，解析 `date`(YYYY-MM-DD)+可选 `time`(HH:MM) 合成 `occurredAt`，用户没提时间就沿用此刻的时分）。首页「记一笔」按钮行为由 `FabActionMode`（`verifin.fab_action.v1`，默认 manual）决定：manual 走数字键盘 + `EntryDetailPage`，ai 走 `ai_entry_sheet.dart` 的自然语言输入 → 解析草稿 → `EntryDetailPage(initialDraft:)` 预填确认（AI 不直接落账），manualTapAiLongPress 则点击手动、长按 AI（首页 FAB 用 `Material`+`InkWell` 自绘、单个 InkWell 同持 `onTap`/`onLongPress`——`FloatingActionButton` 内部 InkWell 会吞掉外层 `GestureDetector` 的长按，故不用它；`shell.dart` 的 `_startQuickEntry(longPress:)` 按手势解析动作）。记账默认付款账户由 `verifin.default_account.v1`（KV，`Map<bookId,accountId>`，每账本各一个、设备本地、不进 JSON 备份、初始化随账户清空、删账户时清引用）承载，`controller.defaultAccountId` 校验账户存在且未隐藏后返回：手动记账把它作为 `EntryDetailPage(initialAccountId:)` 默认账户；AI 未识别到账户（`accountId` 空）时也回落到它，未设则记为「无账户」。设置入口两处：「我的 → 设置 → 默认账户」（账户选择弹窗，可选「无默认」）与账户详情页 `CompactSwitchRow`「设为默认账户」。数字键盘 `NumberPadSheet` 支持四则运算算式（如 `500+800`）：纯函数 `lib/app/calc_expression.dart` 的 `evaluateAmountExpression` 按标准优先级求值（含算式返回 null=不完整），面板右下角浅色展示结果预览，不完整时提示「算式不完整」并禁用确认。`AiSettings` 存 KV（`verifin.ai.v1`，明文本机、不进 JSON 备份、初始化保留），设置页在「我的 → 设置」，子页 `AiSettingsPage`（`ai_settings_page.dart`）配置并测连通性。Android 已有 `INTERNET` 权限，并开启 `usesCleartextTraffic="true"` 以支持本地 / 自建 `http://` 端点（如局域网 Ollama / LM Studio），云端 https 端点同样可用。
- `lib/app/screenshot_recognizer_*.dart`：截图识账的端上 OCR — io 用 `google_mlkit_text_recognition`（中文模型、离线，图片不出设备；按字节识别时落临时文件走 `InputImage.fromFilePath`，识别完即删），stub（测试宿主）不支持。**坑**：该插件对各脚本识别库只 `compileOnly`，App 必须在 `android/app/build.gradle.kts` 显式 `implementation("com.google.mlkit:text-recognition-chinese:...")`，并在 `proguard-rules.pro` 给未引入的梵/日/韩脚本加 `-dontwarn`——否则 debug 正常、**release 的 R8 报缺类构建失败**（CI 才暴露）。**定案：应用永久不做监听类自动记账**（不注册 NLS/无障碍/短信，见 `docs/dev/auto-capture-plan.md`），采集一律由用户主动触发：① 其他 App 分享图片/文本给 Veri Fin；② 自动化工具（Tasker 等）显式发 `top.talyra42.verifin.action.CAPTURE_TEXT`（extra `text`，用户文档 `docs/automation.md`）。两者都进 `ShareReceiverActivity`（无界面跳板：`ACTION_SEND` 会把目标起在分享方任务栈，直指 MainActivity 会双 Flutter 引擎，故转发回自家任务栈后立即 finish），MainActivity 暂存后经 `verifin/app` 通道 `consumeCaptureImage`/`consumeCaptureText` 被拉取（运行中由 `openSharedCapture` 通知，冷启动由 shell 开屏主动 consume，仿快速记账 intent 模式；原生侧文本限 8000 字、图片限 25MB）。Flutter 流程在 `lib/pages/capture_entry.dart`：AI 硬门槛（未配置先引导）→ 模态加载态 → OCR → `ai_entry_parser.dart` 的 `requestCapturedEntryDraft`（`buildCapturedEntryPrompt` 在基础提示词上追加 OCR 噪音/忽略余额/交易时间优先/非交易置 0 规则；`capturedTextLikelyTransaction` 无数字短路不调 AI、超 4000 字截断）→ `EntryDetailPage(initialDraft:)` 确认，**绝不静默落账**；分享图不落库不留存。App 内入口：AI 记账弹层的「截图识账」按钮（相册选图，`pickScreenshotPath`）。

[lib/app/platform_bridge.dart](lib/app/platform_bridge.dart) 是 Android MethodChannel 桥（快速记账磁贴入口、分享/外部采集内容消费（`consumeCaptureImage`/`consumeCaptureText`）、GitHub Release 更新检查与下载（`checkLatestRelease`/`downloadLatestUpdate` 带 `includePrerelease` 参数：稳定通道用 `/releases/latest`，预发布通道拉 `/releases` 列表剔草稿后取版本号最高的一个；结果回传 `isPrerelease` 供下载前风险提示，更新弹窗有「包含预发布版本」开关）、备份目录 SAF 读写、桌面小组件数据推送 `updateWidgetData`、一键固定小组件 `pinWidget`）。桌面小组件有三个：今日支出+记一笔（`QuickEntryWidgetProvider`）、本月可用预算（`BudgetWidgetProvider`）、资产总额（`NetWorthWidgetProvider`，后两者继承 `StatWidgetProvider` 共用 `stat_widget` 布局）；数据经 `WidgetData`（原生 SharedPreferences 读写 + 广播刷新）承载，Flutter 侧 `home_widget_service.dart` 的 `pushWidgetData` 在开屏/回前台/记账后一次推送三者的值（均按用户偏好格式化）。**跨天/跨月自愈**：推送时额外带上「今日支出」对应日期（`todayDate`）+归零值、「本月可用」对应月份（`budgetMonth`）+整月预算值；原生渲染时若锚点日期/月份已过期就自动展示归零值/整月预算（`WidgetData.todayAmountForToday`/`budgetForMonth`），不必等应用打开重新推送。`WidgetRefreshReceiver`+`WidgetRefreshScheduler` 用 `AlarmManager.setAndAllowWhileIdle`+`RTC_WAKEUP`（无需精确闹钟权限）在每晚午夜刷新一次全部小组件触发自愈，并在推送/重绘/开机（`BOOT_COMPLETED`/`MY_PACKAGE_REPLACED`）时重排下一晚闹钟；**必须用 allow-while-idle+wakeup**——普通 `set(RTC,…)` 在 Doze 下整夜被压制、不唤醒设备，会导致午夜闹钟不触发、次日打开手机仍是旧数据（历史 bug）。两个时间敏感小组件（今日支出/本月预算）的 `updatePeriodMillis` 设为 30 分钟作为「设备活跃时」的第二条兜底触发（不唤醒设备、搭已有唤醒周期的车），资产总额不随时间过期故不需锚点也不设周期。国产 ROM force-stop 会取消闹钟且收不到广播，此场景无法在代码内根治，靠用户加自启动白名单或次日开 App 恢复。应用内小组件展示页 `lib/pages/widget_gallery_page.dart`（我的 → 数据与工具 → 桌面小组件）预览各小组件样式、经 `pinWidget` 请求系统「添加到桌面」（启动器不支持时回落手动添加引导）。小组件外观为随系统深浅色切换的白/黑卡片（`res/values(-night)/colors.xml` 的 `widget_*` 颜色 + `widget_background` 描边卡片，不用大面积渐变），强调色仅用于「记一笔」按钮。点「记一笔」复用快速记账 `ACTION_QUICK_ENTRY`。新增小组件类型必须在编译期声明 `AppWidgetProviderInfo` XML + Manifest receiver（运行时无法凭空新建）。

### 数据管理与备份

数据管理页（`DataManagementPage`，我的页进入）集中导出/导入/初始化，以及 `lib/app/backup/` 备份子系统：`backup_settings.dart`（目录/频率/保留/上次时间 + 纯保留清理逻辑）、`backup_archive.dart`（zip 打包/解包纯函数：把图片附件字节从 JSON 剥离、单独存 `attachments/<id>`，与 `backup.json` 一起打进 zip，解包拼回 `dataUrl`；`looksLikeZipBytes` 按魔数识别）、`backup_crypto.dart`（AES-GCM+PBKDF2 加密，`cryptography` 包）、`backup_service.dart`（`prepare` 决定格式：未加密→zip 字节 `.zip`、加密→文本信封 毫秒故无损精确到秒，`assetId` 经 assets 数组映射为账户名，分类取二级分类叶子回退一级，转账 type=2/分类「资产互转」的转出/转入账户编码在 `note`「转出 -> 转入」里、按此拆解，选 zip 而非其 CSV「账单」导出是因后者只有日期无时分；**Tally 还导入 assets 数组里的账户余额与类型**——普通来源只由交易派生账户、余额从流水算出，Tally 走 `buildPlatformImportPlan` 里的专用分支 `_applyTallyAssets`：对本次新建的账户回推 `initialBalance = 目标余额 − Σ交易增量` 使导入后显示余额对齐 Tally 当前余额，没有流水的资产（零余额钱包、借出/负债对象）也直接以当前余额补建；Tally `amount` 恒为正、符号由 type 决定（负债 1/分期 4 记负，资产 0/借出 2/理财 3 记正，与 Veri Fin「余额正负=资产/负债」一致），`isIncludedInTotal`→`includeInAssets`；补建的账户经 `ImportPlan.standaloneAccountIds`→`ImportPreviewResult.alwaysCreateAccountIds` 传给 `applyImportEntries`，使其即便无交易引用也会落库、不被「只建被引用账户」的过滤丢掉），各来源归一到 Veri Fin 规范列后复用 `buildImportPlan`；解析基于用户真实样例，禁止编造格式）、`xls_reader.dart`（最小 OLE2 复合文档 + BIFF8 读取器，纯 `dart:typed_data`、无库依赖：取 `Workbook` 流、SST 含 CONTINUE 跨记录拆分、LABELSST/NUMBER/RK/MULRK 单元格；pub.dev 无 BIFF8 库故自写）、`webdav_config.dart`/`webdav_client_*.dart`（WebDAV）。导入入口为数据管理页「导入账单文件」：**先选平台（支付宝/微信/薄荷/一木·账单/一木·转账还款/Tally/其他 CSV）再选文件**，并附各平台「如何导出账单」引导；平台选择器用真实软件品牌图标（`assets/import_icons/`，与账户图标 `assets/account_icons/` 不同）。**选文件后不直接落库，先进导入预览页**（`lib/pages/import_preview_page.dart`）：`controller.parsePlatformImport` 只解析出 `ImportPlan`，预览页把交易**按日期分组**（复用 `common_widgets` 的 `groupEntriesByDate`/`DateGroupHeader`/`TransactionListCard`）列出，可逐条排除或点进 `EntryDetailPage.draft`（草稿模式，返回改后条目而不落库）完整编辑；顶部可折叠的「导入账户 / 导入分类」映射区可给每个待新建账户/分类改名或映射到现有条目（按待新建 id 整体替换，对所有引用它的交易一次生效）。确认返回 `ImportPreviewResult`，`controller.applyImportEntries` 只创建被保留交易实际引用到的新账户/分类（外加 `alwaysCreateAccountIds` 里携带余额的独立账户，即便无交易也建）。携带余额的来源（Tally）在预览的「导入账户」区默认展开、每行右侧显示导入后余额；**只有账户、没有交易的备份也能导入**（预览页确认按钮显示「确认导入（N 个账户）」，`buildPlatformImportPlan` 不因 `records` 缺省而报错）。预览/编辑传「现有＋待新建」的合并账户/分类列表，全程零落库副作用。**以后新增导入平台一律走此预览流程。**备份目录/加密口令/WebDAV 配置存 KV，均不随初始化数据清除，也不进 JSON 导出（设备本地）。

应用锁（PIN/图案/生物解锁）：加盐哈希（`lib/app/app_lock.dart`）存 KV，UI 与门卫在 `lib/pages/app_lock_page.dart` 与 `app_lock_gate.dart`（门卫放在 `MaterialApp.builder` 覆盖根 Navigator，冷启动与回前台锁定）。

### 页面层

[lib/main.dart](lib/main.dart) 只保留应用入口与根组件。页面按功能拆分在 `lib/pages/`：`shell.dart` 是底部四 Tab 导航壳（首页/资产/看板/我的），`home_page.dart`、`budget_pages.dart`、`transactions_pages.dart`、`assets_pages.dart`、`reports_page.dart`、`profile_pages.dart`、`entry_detail_page.dart` 各自承载一个功能域，跨页面共享的底部弹窗/对话框在 `sheets.dart`。模型（含 JSON 序列化）在 [lib/app/models.dart](lib/app/models.dart)，通用组件在 `lib/app/common_widgets.dart`，底部弹窗在 `lib/app/entry_sheets.dart`，自绘图表及其交互组件（`InteractiveTrendChart`/`InteractiveBarChart`、数据气泡、命中测试）在 `lib/app/chart_painters.dart`，图表序列/坐标轴纯函数在 `lib/app/series_math.dart`，多级分类的树形查询/前序展开纯函数（邻接表 `Category.parentId`，带环检测）在 `lib/app/category_tree.dart`，周期记账的到期日推进纯函数在 `lib/app/recurring.dart`（`main.dart` 在开屏与回前台调用 `controller.applyDueRecurring(now)` 补记到期交易），首页概览卡（`HomeTrendPanel`）可自定义展示数据：指标计算纯函数在 `lib/app/home_metrics.dart`（`computeHomeMetric`/`HomeMetricContext`，21 个指标；`HomeTrendConfig` 存 KV `verifin.home_metrics.v1`，设备本地、不进备份、初始化保留），自定义页 `lib/pages/home_metrics_settings_page.dart`（收支统计页右上角「编辑」进入，点槽位弹窗选指标、可复位），走势窗口为累积展开（`cumulativeWeekWindowFor`，详情页 `IncomeExpenseStatsPage` 支持周/月/季/年视图切换：周·月·季按天窗口 `weekWindowFor`/`monthWindowFor`/`quarterWindowFor`，年按 12 个月聚合 `monthlyNetValuesForType`，年视图下方按月列出、点某月下钻回该月），记账自动识别的推断纯函数（`suggestEntry`：从用户自己的历史里按「金额接近度+备注相似度+时段」找最像的几笔，加权推断当前这笔的**类型/分类/标签/备注**——不只是分类；单笔支撑须是强匹配（精确金额或备注重合）才敢翻转类型，避免松散金额误判）在 `lib/app/category_suggest.dart`。`entry_detail_page.dart` 在开屏（金额已定）、改金额、改类型、输备注时调用，逐字段填充「用户尚未手动改过」的项（`_typeTouched/_categoryTouched/_tagsTouched/_noteTouched`），静默无提示；AI 草稿模式整体关闭自动识别。所有图表都必须支持点击展示数据，规范见 `docs/ui-guidelines.md` 的“图表交互”一节。只有出现明确复用或复杂度上升时再抽取组件。

分类图标：内置图标 code→`IconData` 由 `demo_data.dart` 的 `iconForCode` 映射（`categoryIconCodes` 为可选清单）；**另支持 emoji 自定义图标**，以 `emoji:` 前缀存进 `Category.iconCode`（如 `emoji:🍜`，助手 `isEmojiIconCode`/`emojiOfIconCode`/`emojiIconCode`）。**分类图标一律用 `common_widgets.dart` 的 `CategoryIconBox`（带色块背景）或 `CategoryGlyph`（裸字形，Chip/内联用）渲染**，二者自动区分内置图标与 emoji——新增分类图标渲染点不要直接 `iconForCode(category.iconCode)`，否则 emoji 分类会回退成钱包图标。选择器 `showCategoryIconPickerSheet`（`sheets.dart`）为图标网格 + 常用 emoji 快选 + 自由输入。emoji 走 `iconCode` 存储，无 schema 变更、导出/导入天然覆盖。

统计分析页（`report_analysis_page.dart`）的排行支持三种分组维度（`_ReportGrouping`）：顶级分类（可点行下钻看子分类拆分）、子分类（按记账所选分类，`reportCategoryStatsByOwn`）、标签（`reportTagStats`，一笔计入其每个标签、占比可重叠）；纯函数都在 `report_analysis.dart`。

### 国际化

使用 Flutter 内置 gen-l10n：ARB 文件在 `lib/l10n/`（`app_zh.arb` 为模板 + `app_en.arb`），`flutter pub get` 时自动生成 `AppLocalizations`（生成文件同目录，已提交）。**新增用户可见文案必须写入 ARB（zh + en 同步）并通过 `AppLocalizations.of(context)` 引用，绝不硬编码中文**。存量文案已全部迁移完成（2026-07）：应用支持中英双语，设置页「语言」可选跟随系统/简体中文/English（`LocalePreference` 存 KV `verifin.locale.v1`，设备本地、不进备份、初始化保留），`main.dart` 经 `localePreferenceListenable` 驱动 `MaterialApp.locale`。枚举显示名统一为 `label(AppLocalizations)` 方法；无 BuildContext 场景（桌面小组件、本地通知、生物识别系统弹窗）经 `lib/app/l10n_outside_context.dart` 的 `l10nForPreference` 按偏好解析。种子数据（默认账本/分类/简介）按首启动语言播种（`VeriFinController.create` 的 `systemIsEnglish`），播种后属用户数据不再切换。有意保留中文：法律文档正文、银行/品牌图标名、CSV 模板表头与逐行导入错误、无 context 的网络/桥接错误消息（有意暂缓，未英文化）。widget 测试脚手架预置中文（`makeController` 写 `verifin.locale.v1=zh`），直接 pump 单页的测试用 `zhMaterialApp`。真机验证清单见 `docs/dev/i18n-verification.md`。

## 关键约束

- **Android 功能必须有真实实现**：涉及本地存储、图片、文件、安装包或系统入口的功能，必须确认 Android 端有真实实现，并检查权限（Manifest、分区存储、Android 13+）、持久化和进程重启后的行为。data URL 图片预览用内存图片渲染，不要用 `Image.network`。
- **数据结构变更要同步**：新增本地数据项时，确认导出/导入、设置页初始化、进程重启读取都覆盖到，并更新样例备份 `docs/dev/verifin-sample-backup.json`（测试应能真实导入它）。
- **文档同步**：变更影响开发命令、架构、交付流程、配置或用户可见行为时，须在同一次变更中更新 `README.md`、`AGENTS.md`、`docs/`。
- **CHANGELOG 同步**：凡用户可见的改动（新功能、界面/交互变化、修复、隐私安全项等），在同一次变更中往 `CHANGELOG.md` 的 `## [Unreleased]` 段落加一行（分类到 新增/优化/修复/安全 等小节）；纯内部重构、改测试/文档等对用户无感的不写。发版时再把 `Unreleased` 提升为版本号（见「发布与 CI」）。
- UI 风格：紧凑型移动端工具风格；主色 `#346edb`，辅助蓝 `#3498db`，青绿色仅用于收入/正向状态。UI 规范见 `docs/ui-guidelines.md`。
- **组件化优先**：写任何 widget/弹窗/对话框/格式化/计算前，先查 `docs/dev/components.md` 组件清单，命中就复用或参数化扩展，不新建变体。图标渲染走 `CategoryIconBox`/`AccountIconBox`（勿直接 `iconForCode`），弹窗走 `show*Sheet`/`showConfirmDialog`/`showTextInputDialog`（勿裸包 `showModalBottomSheet`/内联 `AlertDialog`）。完整代码规范见 `AGENTS.md` 的「代码规范 · 组件化与工程约定」。已知架构限制与技术债见 `docs/dev/known-limitations.md`。
- 不要重写生成的平台工程文件（`android/`），除非任务明确要求；不要为简单需求引入额外工具链或依赖。
