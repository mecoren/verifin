# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

Veri Fin (包名 `verifin`) 是一个完全免费、数据自主、本地优先的 Flutter 记账应用。仓库规则详见 `AGENTS.md`（中文），本文件是其摘要加架构说明。**与用户沟通时使用中文。**

## 常用命令

```bash
flutter pub get                        # 安装依赖
flutter run -d chrome                  # 本地预览（默认走 Web 调试）
flutter analyze                        # 静态检查（flutter_lints）
flutter test                           # 运行全部测试
flutter test test/widget_test.dart     # 运行单个测试文件
flutter test --plain-name "关键字"      # 按名称运行单个测试
dart format .                          # 提交前格式化
flutter build web --pwa-strategy=none  # 验收用 Web 产物（禁用 PWA 缓存避免旧版本）
```

验收 Web 产物：构建后在 `build/web` 下运行 `python3 -m http.server 8080 --bind 0.0.0.0`，移动端视口用 390x844 检查。

## 发布与 CI

- **不要在本机构建 APK 作为交付物**。Android 安装包只由 GitHub Actions 生成。
- CI（`.github/workflows/flutter.yml`）只在推送 `vX.Y.Z` 标签时触发：analyze → test → build web → build apk --release → 创建 GitHub Release。普通 `main` 推送不触发。
- 发版用 `scripts/publish.sh patch`（也支持 `minor`、`major` 或显式版本号）；脚本会更新 `pubspec.yaml` 和 `appVersionLabel`、提交、打标签并推送。
- Release APK 必须用项目内稳定 keystore（`android/app/verifin-release.jks`）签名。

## 提交规范

- 提交信息用 `type: summary` 格式（如 `feat: add login screen`、`fix: correct counter test`）。
- **提交中不要包含任何 AI 署名**（`Generated by ...`、`Co-authored-by: ...` 等）——这是仓库明确规则，覆盖默认行为。
- 每完成一个相对独立的模块就提交一次，不要把所有改动混到最后一次提交。

## 架构

### 状态管理：单一 Controller + InheritedNotifier

全部应用状态集中在 [lib/app/veri_fin_controller.dart](lib/app/veri_fin_controller.dart) 的 `VeriFinController`（`ChangeNotifier`）中：交易、账户、分组、账本、分类、预算、个人信息、主题和各种视图偏好。它通过 `main.dart` 中的 `VeriFinScope`（`InheritedNotifier`）注入组件树。没有 Provider/Riverpod/Bloc 等第三方状态库。

- 持久化：每类数据以 JSON 存入 `LocalKeyValueStore`，键形如 `verifin.entries.v1`。
- 多账本隔离：交易、账户、分组都带 `bookId`，controller 的 getter 按 `_activeBookId` 过滤后返回不可变列表。
- 转账（`EntryType.transfer`）同时影响转出/转入账户余额，但不计入收支统计（聚合逻辑在 [lib/app/ledger_math.dart](lib/app/ledger_math.dart)）。

### 平台适配：条件导入三件套

Web/移动/测试差异统一用条件导出模式（`stub` + `if (dart.library.html)` + `if (dart.library.io)`）：

- [lib/local_storage/](lib/local_storage/)：`LocalKeyValueStore` — io 用 `SharedPreferences`，测试用内存 stub。真实平台启动必须 `LocalKeyValueStore.create()`，**非 Web 平台绝不能落到内存实现**。
- `lib/app/avatar_picker_*.dart`：本地图片选择。
- `lib/app/data_file_port_*.dart`：JSON 备份导出/导入（Android 默认写系统 Downloads）。

[lib/app/platform_bridge.dart](lib/app/platform_bridge.dart) 是 Android MethodChannel 桥（快速记账磁贴入口、GitHub Release 更新检查与下载）。

### 页面层

[lib/main.dart](lib/main.dart)（约 9000 行）是刻意的单文件页面层：应用入口、`VeriFinShell` 底部四 Tab 导航（首页/资产/看板/我的）和所有页面 Widget。模型（含 JSON 序列化）在 [lib/app/models.dart](lib/app/models.dart)，通用组件在 `lib/app/common_widgets.dart`，底部弹窗在 `lib/app/entry_sheets.dart`，自绘图表在 `lib/app/chart_painters.dart`。只有出现明确复用或复杂度上升时再抽取组件。

### 测试

全部测试在 [test/widget_test.dart](test/widget_test.dart)，使用内存 stub 存储。测试覆盖用户可见行为（渲染、交互、导航、表单校验）。修改功能后至少运行 `flutter analyze` 和 `flutter test`。

## 关键约束

- **移动端功能不能只实现 Web 路径**：涉及本地存储、图片、文件、安装包或系统入口的功能，必须确认 Android 端有真实实现，并检查权限（Manifest、分区存储、Android 13+）、持久化和进程重启后的行为。移动端 data URL 图片预览用内存图片渲染，不要用 `Image.network`。
- **数据结构变更要同步**：新增本地数据项时，确认导出/导入、设置页初始化、进程重启读取都覆盖到，并更新样例备份 `docs/dev/verifin-sample-backup.json`（测试应能真实导入它）。
- **文档同步**：变更影响开发命令、架构、交付流程、配置或用户可见行为时，须在同一次变更中更新 `README.md`、`AGENTS.md`、`docs/`。
- UI 风格：紧凑型移动端工具风格；主色 `#346edb`，辅助蓝 `#3498db`，青绿色仅用于收入/正向状态。UI 规范见 `docs/ui-guidelines.md`，参考图在 `docs/targetImages/`。
- 不要重写生成的平台工程文件（`android/`、`web/`），除非任务明确要求；不要为简单需求引入额外工具链或依赖。
