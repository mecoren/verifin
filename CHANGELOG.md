# 更新日志

本文件记录 Veri Fin 面向用户的版本变化，最新版本在前。仅收录用户可见的改动，纯内部重构 / 文档更新不在此列。

改动随做随记：做了用户可见的改动就往 `## [Unreleased]` 加一行；发版时把 `Unreleased` 改名为版本号加日期，再在其上新开一个空的 `Unreleased`。

## [Unreleased]

### 新增
- AI 记账设置页支持**清空配置**：右上角「清空配置」按钮，确认后清空请求地址 / API Key / 模型（此前填了配置后无法一键清空）。

### 优化
- **AI 助手页面整体翻新**：头部改用与其它页一致的通用样式；输入框和发送按钮加大、发送按钮改为圆形并在无输入时置灰不可点；聊天区左右间距、消息字号与行距、图表坐标轴与表格样式全面打磨，更精致。

### 修复
- AI 助手退出再进入后，之前对话里的**图表 / 结果卡片丢失**——现在结果卡片会随聊天记录一起保存，重开时连同图表一并还原。

### 新增
- **左右滑动切换主页面**：首页 / 资产 / 看板 / 我的 四个 Tab 之间可左右滑动切换，不必只点底部按钮。在图表或可滑动删除的交易行上拖动仍是原来的交互（选数据点 / 滑动删除），只有在空白区滑动才切换页面。
- **AI 对话查询账目**：看板页右上角新增「问 AI」入口，进入全屏聊天页，用自然语言问账目（如「这个月花最多的是哪些分类」「最近三个月有哪些大额支出」「本月收支怎么样」）。AI 会调用只读工具查询你**当前账本**的真实数据，以柱状图 / 折线图 / 可点击的交易列表 + Markdown 文字（支持表格）作答，答复流式逐字呈现；点结果里的交易可直接进详情页。用的是你在「AI 记账设置」里配置的服务，未配置时页面会引导去配置。聊天记录**仅存本机**、不进备份，右上角可一键清空。AI 全程只读，不会改动任何账目数据。

## [1.10.7] - 2026-07-11

### 优化
- 退款金额输入：数字键盘里直接显示「最多 XXX（剩余可退）」，超上限时点确认当场封顶——不用等保存才知道超了。
- 退款编辑弹窗的「删除」挪到右上角，与底部「保存」分开，避免误触。

## [1.10.6] - 2026-07-11

### 新增
- **退款升级为独立条目**：一笔支出可添加**多笔退款**，每笔各记**金额 / 到账账户 / 发起日期 / 到账日期 / 备注**，支持全额 / 部分退款（不超过原额，超出会提示）。在交易详情页的「退款」区添加、编辑、删除，可带图片附件。
- **待到账 / 已到账两态**：退款可先记为「待到账」（已申请、钱还没回来，不影响余额与统计），到账后一键「标记已到账」核销。交易列表页头新增「**待退款**」清单，汇总所有在路上的退款、显示合计、可逐笔一键核销。
- 退款可**退到与原支付不同的账户**（如原卡已注销、平台退到余额）。

### 优化
- 退款回款不再计入收入、也不虚增花费：只有「已到账」才冲减对应支出的净额、并把钱加到到账账户，账户余额与收支统计更准确。旧版的单笔退款金额在升级后自动转成一条退款记录，数据无损。

## [1.10.5] - 2026-07-11

### 优化
- 账户详情页的设置项**分组展示**（基本信息 / 卡片信息 / 信用 / 展示与记账 / 危险操作），像设置页一样按模块分区，不同账户类型只显示相关模块，选项多时更清晰;删除账户入口标为红色。
- 账户详情里「卡号后四位」入口改名为「卡号」——点进去本就是填完整卡号 + 后四位,原名容易误解。

### 修复
- 信用卡 / 信用账户**还款后**,首页「最近交易」里那笔还款曾错误显示为「已删除分类」(点进去其实是转账)。现在还款按正常转账分类记录,列表显示正确。
- 用数字键盘设置**信用额度 / 预算**等可为 0 的金额时,按 **C 清空**后「确定」按钮点不了、必须先输个 0 才能确认。现在清空后即可直接确认为 0(用于清除额度 / 预算)。

## [1.10.4] - 2026-07-11

### 新增
- 新增**信用账户**账户类型（位于「网络支付」与「信用卡」之间），适配花呗 / 白条等有额度、账单日、还款但没有实体卡号的信用类账户。
- 信用卡 / 信用账户新增**信用额度**：在账户详情页设置总额度后，直观展示已用额度、可用额度与额度使用进度条；设了账单日的还展示**本期账单**（当前账单周期消费，附账单日 / 还款日提示）。资产列表里信用类账户也会显示「可用额度」。
- 信用卡 / 信用账户新增**还款**入口：账户详情页点「还款」即可发起还款，金额默认预填当前欠款、可改，扣款账户可自由选择、也可选「无账户（代还）」适配他人代还；还款按转账记账、不计入收支统计。
- 信用卡 / 储蓄卡新增**完整卡号**（选填）录入：可填完整卡号，账户详情页展示并支持一键复制，方便转账时粘贴。后四位默认「跟随完整卡号」自动取末四位，也可关掉开关手动填写；**开关状态会被保存并随备份导出 / 导入**，关掉后重新打开或换机导入都保持你的选择，不会又变回跟随；列表 / 首页仍只显示后四位。

## [1.10.2] - 2026-07-10

### 优化
- 收支统计页的走势图新增**周 / 月 / 季 / 年**视图切换：顶部可在四种时间跨度间切换查看支出/收入/转账走势（周·月·季按天描点，年按 12 个月描点），前后翻页按当前档位步进；年视图下方按月列出，点某月即可下钻回该月视图。此前只能按整月查看、切月后总回到当月。

### 修复
- 导入一木记账账单时，带**退款**的支出现在会正确按「金额 − 退款」记净额（退款金额回到原账户），此前「退款」这一列被忽略、支出记成了未扣退款的毛额。

## [1.10.1] - 2026-07-10

### 修复
- 桌面小组件跨天不刷新：此前「今日支出 / 本月可用预算」小组件在你不打开 App 时，次日仍停留在昨天的数据，必须打开一次应用才更新。原因是午夜自愈闹钟在手机夜间休眠（Doze）时被系统压制、整夜不触发。现改为可穿透休眠并唤醒设备的午夜闹钟，并加了「设备活跃时」的兜底刷新，跨天/跨月后无需打开 App 也会自动归零 / 回到整月预算。（国产 ROM 若把应用彻底清理后台仍可能失效，建议给应用加自启动 / 后台白名单。）
- 修复「统计分析 → 分类排行」里同一个分类（如「餐饮」）偶尔会分裂成两行、金额与笔数对不上，而分类管理里其实只有一个的问题。根因是历史 / 异构备份可能带入「指向已删除分类的交易」或「重复的同名分类」。现在：① 分类排行不再把这类异常引用冒名成同名分类，而是显示为「已删除分类」；② 应用启动 / 导入备份时会自动做一次分类数据自愈——把指向已删除分类的交易归入「未分类」、把重复的同名分类合并（交易一并迁移）、把父分类已丢失的子分类重挂为顶级；③ 新增分类唯一性约束，从源头防止再产生重复同名分类。

## [1.9.2] - 2026-07-09

### 优化

- 账户分组选择图标时,改为像账户图标一样可直接看到图标预览,不再是只有文字名称的列表。

- 交易列表按分类筛选,改用与记账一致的分类选择器(带图标、可折叠的父子层级),顶部提供「全部」项,不再是纯文字加空格缩进的列表。

### 修复
- 删除账户时,原本引用该账户的周期记账规则会被自动停用并清空其账户(改为「无账户」),并提示你前往复查,不再持续往已删除的账户补记流水。
- 分类若正被周期记账规则引用,将无法删除并给出提示,避免规则到期后生成「归不了类」的交易。
- 切换到后台时一并把刚记下的交易落盘(此前仅刷新偏好设置),消除极端情况下「刚记的账被系统回收丢失」的隐患。
- 数字键盘算式记账(如 `0.1+0.2`)结果规整到「分」,不再把 `0.30000000000000004` 这类浮点尾差存进账目。
- 导入 Tally 备份的预览页,账户「导入后余额」改为随你排除 / 修改交易实时重算,确保预览所见与实际导入结果一致(此前一直显示 Tally 原始余额,排除交易后会对不上)。
- 导入 Tally 备份时,同名的多个资产账户(如两个都叫「现金」的钱包)不再被合并成一个,而是各自建立、余额各归各的(重名的自动加「 (2)」后缀区分)。

## [1.8.7] - 2026-07-07

## [1.8.6] - 2026-07-07

### 修复
- 修复**把一笔待报销支出的金额改到低于已报销/退款额**后，这笔支出会被错误计入「收入」、账户余额虚增的问题：净额现在钳制为不小于 0。
- 数据库因损坏或**降级安装了更旧版本**而打不开时，不再白屏卡死，改为显示明确的错误页——提示数据很可能仍在本机、**请勿清除应用数据**，并给出重装最新版的引导；绝不会自动删库。
- **恢复备份 / 重置数据 / 删除账本**现在在单个数据库事务内完成：万一中途进程被杀或写入出错，会整体回滚，不再可能留下「交易换了但账户没换」的半新半旧状态。
- WebDAV 请求增加连接与响应超时：当服务器无响应时不再无限期挂起，此前这会导致**自动备份此后被静默跳过、一直不再运行**（需重启应用才恢复）。
- 应用切到后台时刷盘挂起的设置写入，缩小「刚设完应用锁 / 刚同意隐私政策就被系统回收 → 设置丢失」的窗口。
- 导入账单时遇到不存在的日期（如 2 月 30 日）或越界时间，不再被静默改成邻近日期，而是按错误行提示、不误记。
- 备份 / 导入失败的提示改用本地化文案，不再向英文用户显示中文原文或原始技术异常信息（空文件、格式无效等有了清晰提示）。
