# Veri Fin 产品文档

## 产品定位
Veri Fin 是一个完全免费、数据自主、本地优先的记账工具。短期内不提供云同步，用户的账本、账户和交易记录优先保存在本地环境中。一期目标不是完成全部记账能力，而是先打通可预览、可保存、可继续扩展的基础记账流程。

## 一期范围
一期实现 4 个底部 Tab 页面：`首页`、`资产`、`看板`、`我的`。整体视觉参考 `docs/targetImages`，主色板使用 `#34dbcb`、`#34c2db`、`#3498db`、`#346edb`、`#3445db`，其中 `#3498db` 是主要交互色，`#346edb` 和 `#3445db` 用于蓝色层级，青绿色仅作为辅助点缀。应用默认跟随系统深浅色，并在“我的”页提供主题切换。

当前视觉方向是紧凑型移动端工具界面：底部导航仅保留图标，小圆角卡片和按钮优先，Header 与分区标题使用较低字号，列表行保持较高信息密度。首页保留浅色工具界面，趋势面板采用深色科技感信息卡；交易列表和交易详情采用深色沉浸风格，以贴近参考图里的查账体验。后续新增页面应沿用这一套视觉令牌，避免大圆角、大标题和营销式卡片堆叠。

不在一期实现多账本管理、云同步、项目、商家、标签、退款等扩展字段。Android 安装包不在本机打包，后续由 GitHub CI 负责生成。

## 核心流程
用户在首页点击右下角 FAB 后，从底部弹出数字键盘。输入金额并点击 `OK` 后进入记账详情页。详情页支持选择 `支出 / 收入 / 转账`、修改金额、选择分类、选择账户、输入备注，并保留日期和时间。

点击保存后，详情页关闭并返回首页，刚保存的交易出现在首页交易列表中。首页默认展示最近 5 条记录，点击交易卡右上角入口进入全部交易列表。交易列表支持按日期分组展示，并提供全部时间、日期降序等筛选入口占位。刷新 Web 页面后，交易记录仍应保留。

点击单条交易进入交易详情页。详情页保留金额、类型、分类、账户、备注、日期、时间等当前数据结构字段，并提供删除按钮；删除交易前必须弹出确认对话框。不在本阶段实现交易编辑。

## 页面说明
首页展示月度支出趋势、最近交易、预算概览和带月份切换的日历视图。资产页展示净资产卡片、账户分组和账户列表，并支持添加账户、管理分组和查看账户详情。记账详情页的账户选择来自本地账户数据。看板页展示分类统计、日趋势和月度收支结构，优先使用已保存记录聚合。我的页展示用户信息、基础统计、账本入口和本地数据状态，主题切换放在设置页。

## 数据策略
一期使用浏览器本地存储保存交易记录、账户、账户分组、个人信息和主题偏好。最小交易数据模型包含交易类型、金额、分类、账户、备注、日期和时间。账户数据包含类型、名称、余额、图标、备注、分组、计入资产和隐藏状态。后续如果迁移到 SQLite、文件存储或移动端持久化，需要保持本地优先和无强制云同步原则。

## 技术结构
Flutter 入口保留在 `lib/main.dart`，基础模型、状态控制、主题、示例数据、记账工具函数、底部弹窗和通用组件拆分到 `lib/app/`。Web 预览以移动端宽度为主，避免桌面宽屏下界面过度拉伸。
