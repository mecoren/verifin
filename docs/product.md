# Veri Fin 产品文档

## 产品定位
Veri Fin 是一个完全免费、数据自主、本地优先的记账工具。短期内不提供云同步，用户的账本、账户和交易记录优先保存在本地环境中。一期目标不是完成全部记账能力，而是先打通可预览、可保存、可继续扩展的基础记账流程。

## 一期范围
一期实现 4 个底部 Tab 页面：`首页`、`资产`、`看板`、`我的`。整体视觉参考 `docs/targetImages`，主色板使用 `#34dbcb`、`#34c2db`、`#3498db`、`#346edb`、`#3445db`，其中 `#346edb` 是主要交互色，`#3445db` 用于深蓝层级，`#3498db` 作为辅助蓝，青绿色仅用于收入和正向状态。应用默认跟随系统深浅色，并在设置页提供主题选择器。

当前视觉方向是紧凑型移动端金融工具界面：底部导航仅保留图标，选中态使用主蓝色胶囊背景；小圆角卡片和按钮优先，页面顶部统一使用可配置的 Header 组件，支持返回、标题、副标题和右侧操作；Header 与分区标题使用较低字号，列表行保持较高信息密度。首页采用浅色工具界面和轻量数据卡片，趋势图保留坐标、网格、日期标签和数据点，并独立在浅色绘图区内；预算卡使用自绘圆环，日历卡使用稳定高度的日期格。首页趋势卡和预算卡只保留点击跳转，不使用长按高亮卡片态。金额为 0 时统一显示为中性色 `0`，不展示 `-0`。快速记账数字键盘限制为移动端宽度，并保持 4:3 按键比例，按钮使用统一弱背景。交易列表和交易详情采用深色沉浸风格，以贴近参考图里的查账体验。后续新增页面应沿用语义色、轻阴影和 4/8pt 间距节奏，避免大圆角、大标题和营销式卡片堆叠。

不在一期实现云同步、项目、商家、标签、退款等扩展字段。Android 安装包不在本机打包，后续由 GitHub CI 负责生成。

## 核心流程
用户在首页点击右下角 FAB 后，从底部弹出数字键盘。输入金额并点击 `OK` 后进入记账详情页。详情页支持选择 `支出 / 收入 / 转账`、修改金额、选择分类、选择账户、输入备注，并保留日期和时间。

点击保存后，详情页关闭并返回首页，刚保存的交易出现在首页交易列表中。首页默认展示最近 5 条记录，点击交易卡右上角入口进入全部交易列表。交易列表支持按日期分组展示，并提供时间筛选和排序弹窗。时间筛选包含全部时间、本年、本季、本月、本周、近12个月、近30天、近6周；排序包含日期降序、日期升序、金额降序、金额升序。刷新 Web 页面后，交易记录仍应保留。

首页支出趋势卡按 7 天周期展示，点击后进入收支统计页。统计页支持切换月份和交易类型，并按当前 7 天周期展示每日金额、占比和交易笔数。日历视图点击具体日期后进入当天交易列表，顶部日期可左右切换到前一天或后一天，也可切换回全部时间、本月等普通筛选。

点击单条交易进入交易详情页。详情页保留金额、类型、分类、账户、备注、日期、时间等当前数据结构字段，支持编辑并通过右上角保存按钮提交；删除交易前必须弹出确认对话框。

## 页面说明
首页展示月度支出趋势、最近交易、预算概览和带月份切换的日历视图。预算概览点击后进入预算页，可切换月份、查看本月支出、剩余额度、剩余日均、预算进度和状态提示，并设置该月预算金额和支出分类预算；当分类预算达到 85% 或已超支时，首页预算卡展示最需要关注的分类提醒。资产页展示净资产卡片、账户分组和账户列表，并支持添加账户、管理分组、查看账户详情、账户报告和账户交易筛选；净资产卡片默认使用普通卡片背景，点击右上角图片图标后可更换背景，支持预设线上图片、自定义图片链接、本地图片和清除背景，本地图片会进入统一裁剪页，按卡片比例裁剪压缩后保存。新用户和初始化后的账本默认没有资产账户，账户与账户分组按当前账本隔离。账户详情可编辑类型、名称、图标、备注、分组、计入资产和隐藏状态；调整余额时不会直接覆盖历史数据，而是按差额生成“余额调整”收入或支出交易。删除已有交易的账户时会阻止删除并提示用户隐藏账户，隐藏账户在资产页底部入口和独立隐藏账户页面中集中展示。记账详情页的账户选择来自当前账本内未隐藏的账户数据。看板页展示分类统计、分类排行、7 天日趋势和带纵轴的月度收支结构，分类排行、金额和占比均使用已保存记录聚合。我的页展示用户信息、基础统计、账本入口、分类管理入口和本地数据状态；分类支持按支出、收入、转账管理，可新增、重命名、换图标和拖动排序，已有交易使用的分类不能删除。个人头像编辑复用同一裁剪页，账本支持新增、切换、重命名和删除非默认账本，默认账本可重命名但不能删除。主题切换、数据导出/导入和初始化数据放在设置页，初始化会清空本地交易、账户、账本、分类、预算、个人信息、资产卡片背景和主题偏好。日期和时间选择器使用中文本地化。

## 数据策略
一期使用浏览器本地存储保存交易记录、账户、账户分组、账本、分类、月份预算、分类预算、个人信息、资产卡片背景和主题偏好。设置页支持导出 JSON 本地备份，也支持从 JSON 备份导入恢复；导入会替换当前本地数据，因此必须先二次确认。最小交易数据模型包含账本、交易类型、金额、分类、账户、备注、日期和时间。分类数据包含类型、名称和图标编码，交易只保存分类 id，因此删除分类前必须确认没有历史交易引用。分类预算按月份和分类 id 保存，分类预算为 0 或清空时视为未设置。账户和账户分组均包含账本归属，旧数据缺失账本归属时迁移到默认账本。账户数据包含类型、名称、余额、图标、备注、分组、计入资产和隐藏状态。余额调整通过交易流水表达差额，避免直接覆盖历史记录。后续如果迁移到 SQLite、文件存储或移动端持久化，需要保持本地优先和无强制云同步原则。

## 技术结构
Flutter 入口保留在 `lib/main.dart`，基础模型、状态控制、主题、示例数据、记账工具函数、底部弹窗和通用组件拆分到 `lib/app/`。Web 预览以移动端宽度为主，避免桌面宽屏下界面过度拉伸。
