# Veri Fin 产品文档

## 产品定位
Veri Fin 是一个完全免费、数据自主、本地优先的记账工具。短期内不提供云同步，用户的账本、账户和交易记录优先保存在本地环境中。一期目标不是完成全部记账能力，而是先打通可预览、可保存、可继续扩展的基础记账流程。

## 一期范围
一期实现 4 个底部 Tab 页面：`首页`、`资产`、`看板`、`我的`。整体视觉参考 `docs/targetImages`，主色板使用 `#34dbcb`、`#34c2db`、`#3498db`、`#346edb`、`#3445db`，其中 `#346edb` 是主要交互色，`#3445db` 用于深蓝层级，`#3498db` 作为辅助蓝，青绿色仅用于收入和正向状态。应用默认跟随系统深浅色，并在设置页提供主题选择器。

当前视觉方向是紧凑型移动端金融工具界面：底部导航仅保留图标，选中态使用主蓝色胶囊背景；小圆角卡片和按钮优先，页面顶部统一使用可配置的 Header 组件，支持返回、标题、副标题和右侧操作；Header 与分区标题使用较低字号，列表行保持较高信息密度。首页采用浅色工具界面和轻量数据卡片，趋势图保留坐标、网格、日期标签和数据点，不额外套浅色绘图区背景；预算卡使用自绘圆环，日历卡使用稳定高度的日期格。首页趋势卡和预算卡只保留点击跳转，不使用长按高亮卡片态。金额为 0 时统一显示为中性色 `0`，不展示 `-0`。快速记账数字键盘限制为移动端宽度，并保持 4:3 按键比例，浅色模式使用浅底深字，深色模式使用深底浅字；数字键默认启用系统触感反馈，可在设置页关闭。交易列表和交易详情采用深色沉浸风格，以贴近参考图里的查账体验。后续新增页面应沿用语义色、轻阴影和 4/8pt 间距节奏，避免大圆角、大标题和营销式卡片堆叠。

不在一期实现云同步、项目、商家、标签、退款等扩展字段。Android 安装包不在本机打包，后续由 GitHub CI 负责生成。

## 核心流程
用户在首页点击右下角 FAB 后，从底部弹出数字键盘。输入金额并点击 `OK` 后进入记账详情页。详情页支持选择 `支出 / 收入 / 转账`、修改金额、选择分类、选择账户、转账时选择转入账户、输入单行备注，并保留日期和时间。

点击保存后，详情页关闭并返回首页，刚保存的交易出现在首页交易列表中。首页默认展示最近 5 条记录，点击交易卡右上角入口进入全部交易列表。交易列表支持按日期分组展示，并提供时间筛选、关键词搜索、账户筛选、分类筛选和排序弹窗。关键词可匹配备注、分类、账户、类型和金额；时间筛选包含全部时间、本年、本季、本月、本周、近12个月、近30天、近6周；排序包含日期降序、日期升序、金额降序、金额升序。刷新 Web 页面后，交易记录仍应保留。

首页支出趋势卡按 7 天周期展示，点击后进入收支统计页。统计页支持切换月份和交易类型，并按当前 7 天周期展示每日金额、占比和交易笔数。日历视图点击具体日期后进入当天交易列表，顶部日期可左右切换到前一天或后一天，也可切换回全部时间、本月等普通筛选。

点击单条交易进入交易详情页。详情页保留金额、类型、分类、账户、转入账户、备注、日期、时间等当前数据结构字段，支持编辑并通过右上角保存按钮提交；删除交易前必须弹出确认对话框。转账记录只影响账户余额，不计入收入或支出统计。Android 端可通过系统下拉快捷开关“快速记账”直接唤起应用并打开同一个数字键盘流程。用户在首页按系统返回键时先提示“再次返回退出程序”，短时间内再次返回才退出。

## 页面说明
首页展示月度支出趋势、最近交易、预算概览和带月份切换的日历视图。首页和看板页底部各有一个“n个面板”小字入口，进入对应的面板管理页：每个面板一行，展示名称、灰色描述和开关，关闭的面板不在页面上渲染；点击右上角排序图标进入排序模式，拖动手柄调整面板顺序，再点击右上角确认退出；每页至少保留一个开启的面板，关闭最后一个时用提示阻止；管理页右上角提供恢复默认入口，确认后恢复默认顺序并开启全部面板。预算概览点击后进入预算页，可切换月份、查看本月支出、近 6 月趋势、上月对比、最近 12 个月历史汇总、剩余额度、剩余日均、预算进度和状态提示；本月预算通过顶部“本月可用预算”旁的编辑图标弹窗设置，支出分类预算通过分类行设置；当分类预算达到 85% 或已超支时，首页预算卡展示最需要关注的分类提醒。资产页默认使用类型视图展示净资产卡片、账户分区和账户列表，并支持添加账户、管理分组、分类/类型视图切换、分区展开折叠、分区排序模式、分区内账户长按排序、查看账户详情、账户报告和账户交易筛选；有两个及以上分区时，点击“排序”进入分区排序模式，此时所有分区先折叠为标题并显示拖动手柄，可稳定拖动调整分区顺序（不再因展开高度不同导致错位或卡顿），点击“完成”退出排序模式；分区右侧总额只统计计入资产的账户，无账户时展示添加资产的空状态，账户分组为空时展示创建提示。账户图标支持通用图标和 `assets/account_icons/` 中的 SVG 图标，选择器使用底部列表样式；输入账户名称时会自动匹配银行或支付平台图标，信用卡和储蓄卡可填写卡号后四位，并在资产列表账户名后展示灰色标记。净资产卡片默认使用普通卡片背景，点击右上角图片图标后可更换背景，支持预设线上图片、自定义图片链接、本地图片和清除背景，本地图片会进入统一裁剪页，按资产卡片封面比例裁剪压缩后保存；用户设置背景图后只叠加轻量黑色可读性遮罩，不叠加蓝色品牌渐变；头像和资产背景的本地 data URL 使用内存图片渲染，避免移动端黑屏。新用户和初始化后的账本默认没有资产账户，账户与账户分组按当前账本隔离。账户详情可编辑类型、名称、图标、单行备注、分组、计入资产和隐藏状态；调整余额时不会直接覆盖历史数据，而是按差额生成“余额调整”收入或支出交易。删除已有交易的账户时可选择隐藏账户，或删除账户并同步删除所有相关交易；转账记录会按转出和转入任一账户关联删除。记账详情页的账户选择来自当前账本内未隐藏的账户数据。看板页展示预算执行、分类统计、分类排行、7 天日趋势和带纵轴的月度收支结构，分类统计圆环展示支出金额最高的 5 个分类，剩余分类合并为“其他”；分类排行、金额和占比均使用已保存记录聚合。我的页展示用户信息、基础统计、账本入口和分类管理入口，不再展示本地数据/云同步卡片；个人信息支持头像、昵称、简介、性别、生日、城市和职业。分类支持按支出、收入、转账管理，可新增、重命名、换图标和拖动排序，已有交易使用的分类不能删除。个人头像编辑复用同一裁剪页，账本支持新增、切换、重命名和删除非默认账本，默认账本可重命名但不能删除。主题切换、触感反馈、检查 GitHub Release 更新、数据导出/导入、查看《隐私政策》《用户协议》和初始化数据放在设置页；检查更新会弹窗展示当前版本、最新版本和下载进度，Android 默认导出到系统下载目录；初始化需要双重确认，确认后会清空本地交易、账户、账本、分类、预算、个人信息、资产卡片背景、资产视图偏好、首页与看板的面板配置、触感偏好和主题偏好，但保留隐私政策同意标记。首次启动会弹出不可绕过的《隐私政策》《用户协议》同意弹窗，点击「不同意并退出」即退出应用，同意后持久化不再询问。本地图片裁剪的耗时计算在后台执行，确认后展示加载对话框，避免界面卡顿。日期和时间选择器使用中文本地化。

## 数据策略
账目类核心数据（交易记录、账户、账户分组、账本、分类、月份预算、分类预算）只存本地 SQLite（`sqflite`，Web 走 `sqflite_common_ffi_web` 的 wasm，测试走 ffi）；全新数据库首次启动会播种默认账本与分类。个人信息、资产卡片背景、资产视图模式、资产分区折叠状态、资产分区排序、分区内账户排序、首页与看板的面板开关及顺序、触感偏好、主题偏好和隐私政策同意标记等偏好类小数据仍存键值存储（Web localStorage，Android/iOS `SharedPreferences`）。不允许使用进程内存作为真实运行环境的数据源。设置页支持导出 JSON 本地备份，也支持从 JSON 备份导入恢复；导入会替换当前本地数据，因此必须先二次确认。最小交易数据模型包含账本、交易类型、金额、分类、账户、可选转入账户、备注、日期和时间。分类数据包含类型、名称和图标编码，交易只保存分类 id，因此删除分类前必须确认没有历史交易引用。分类预算按月份和分类 id 保存，分类预算为 0 或清空时视为未设置。账户和账户分组均包含账本归属，旧数据缺失账本归属时迁移到默认账本。账户数据包含类型、名称、余额、图标、单行备注、分组、计入资产、隐藏状态和可选卡号后四位。个人资料包含头像、昵称、简介、性别、生日、城市和职业。转账通过同一条交易记录表达转出和转入账户，余额调整通过交易流水表达差额，避免直接覆盖历史记录。Android 包名为 `top.talyra42.verifin`。存储引擎为 SQLite（开发期直接切换，不兼容旧 KV 数据结构），对外备份交换格式仍为 JSON（老 JSON 备份始终可导入）；后续任何存储或同步能力都需保持本地优先和无强制云同步原则。

## 技术结构
Flutter 入口保留在 `lib/main.dart`，基础模型、状态控制、主题、示例数据、记账工具函数、底部弹窗和通用组件拆分到 `lib/app/`。Web 预览以移动端宽度为主，避免桌面宽屏下界面过度拉伸。
