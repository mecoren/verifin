# Veri Fin 验收清单

## 当前已完成
- 首页支持快速记账、最近交易、收支统计入口、预算设置和日历日期交易入口。
- 预算页支持月份切换、月份预算设置（顶部“本月可用预算”旁的编辑图标弹窗）、分类预算设置、近 6 月趋势、上月历史对比、最近 12 个月历史汇总、支出进度、剩余额度、剩余日均、分类预算排序和状态提示。
- 交易支持本地保存、刷新保留、详情查看、编辑保存和删除确认。
- 资产页默认按账户类型展示，支持账户管理、分组管理、分类/类型视图切换、分区展开折叠、分区排序模式、分区内账户长按排序、账户详情、账户报告和资产卡片背景自定义。
- 新用户和初始化后默认无资产账户；切换账本后账户和分组按账本隔离。
- 账户详情支持余额调整，按差额生成“余额调整”交易记录。
- 隐藏账户集中在资产页底部入口和独立隐藏账户页面中展示。
- 图片入口使用统一裁剪页：头像为 1:1，资产背景为与净资产卡片一致的宽幅封面比例。
- 看板页基于本地交易数据展示预算执行、分类、排行、7 天趋势和带纵轴的月度结构。
- 首页和看板底部提供“n个面板”入口，面板管理页支持开关面板与排序模式拖动调整顺序，配置持久化并随备份导出/导入。
- 我的页支持个人资料编辑、账本管理和设置入口，不展示本地数据/云同步状态卡片。
- 分类管理支持按支出、收入、转账维护分类，并能新增、重命名、更换图标和拖动排序。
- 设置页支持主题选择器、触感反馈开关、手动检查 GitHub Release 更新和版本展示。
- 我的页「数据管理」入口进入数据管理页，集中 JSON 备份导入导出和初始化数据。
- 数据管理页可选择备份目录（Android 弹系统文档树授权、桌面弹目录选择），「立即备份」写入该目录并更新「上次备份时间」；可清除目录授权。
- 选择备份目录后出现自动备份设置：频率（仅手动/每次打开/每次记账后/每隔 N 小时）、间隔与保留份数；自动备份在打开、回前台或记账后按频率触发，超出保留份数的旧自动备份被清理。
- 备份加密：设置密钥后导出/备份文件为密文，导入加密文件需正确密钥（已存密钥自动尝试、失败可手动输入且可重试）；清除密钥需确认，旧加密文件仍需原密钥。
- CSV 导入：可下载模板；导入按模板列把交易追加到当前账本，匹配不到的账户/分类按名称新建，非法行被跳过并列出行号原因；钱迹/随手记等 CSV 能被自动识别导入并提示来源。
- WebDAV：可填写地址/账号/密码并「测试连接」，手动「上传到 WebDAV」，「从 WebDAV 恢复」能列出远端备份并选择恢复（加密文件走解密流程）；开启自动上传后随自动备份一并上传。
- Android 端支持系统下拉快捷开关“快速记账”，点击后进入应用并打开快速记账数字键盘。
- Android 端使用持久化本地存储，应用从后台划掉后重新进入仍应保留交易、资产和设置。
- 选择器统一使用底部弹出样式，破坏性操作使用确认弹窗。

## 验收重点
- 浅色和深色模式下，首页、资产、看板、我的四个 Tab 不应出现白屏、文本重叠或明显低对比内容。
- 首页卡片标题使用统一字号和字重，信息密度应接近“日历”卡片。
- 首页和收支统计的趋势图应按 7 天周期展示，不直接展示整月横轴。
- 首页支出走势图不应额外出现浅色背景框。
- 预算页修改月份预算后，首页预算卡应同步展示新的预算金额和剩余状态；修改分类预算后，预算页对应分类行应立即展示预算金额和进度，临近或超支分类应排在前面。
- 分类预算达到 85% 或已超支时，首页预算卡应展示最需要关注的分类提醒；正常状态不展示额外提醒。
- 预算页历史对比应展示本月支出、上月支出和预算使用率变化；分类预算行应展示上月同分类支出。
- 预算页近 6 月趋势应展示支出柱、预算线、月份标签和纵轴刻度；预算历史页应展示最近 12 个月汇总并可点击月份进入对应预算页。
- 看板页顶部应展示预算执行摘要，包括本月预算、本月支出、分类预算数量和超支分类数量。
- 看板日趋势应与首页使用一致的 7 天周期窗口。
- 面板管理页关闭某个面板后，对应页面不再渲染该模块；调整顺序后页面按新顺序展示；只剩一个开启面板时继续关闭应弹出提示并保持开启；重启应用后配置仍然生效。
- 面板管理页右上角“恢复默认”应先弹出确认框，确认后恢复默认顺序并开启全部面板，取消则保持现状。
- 资产页设置背景图后，展开/折叠账户分区时净资产背景图不应闪烁；选择本地图片确认裁剪时应显示加载对话框，完成后自动关闭。
- 看板分类圆环和月度收支图不应出现文本重叠；分类统计圆环应横向居中，分类名称通过同色引线从圆环指出，不在右侧放最高分类说明；分类统计应展示金额最高的 5 个分类，剩余分类合并为“其他”；月度收支图应显示纵轴刻度。
- 快速记账流程应完整闭环：FAB 输入金额、进入详情、保存后首页出现记录。
- 快速记账数字键盘普通按钮使用统一弱背景，不显示蓝色边框；浅色模式为浅底深字，深色模式为深底浅字，`OK` 可点击时使用主蓝色背景和白色文字，不可点击时有清晰的浅蓝灰禁用态。
- 浅色模式下快速记账数字键盘应为浅色按钮和深色数字，不应出现黑底黑字。
- 首页按系统返回键时应先提示“再次返回退出程序”，短时间内再次返回才退出。
- 日期和时间选择器应显示中文界面。
- 交易详情页应支持修改金额、类型、分类、账户、备注、日期和时间，并可保存。
- 交易详情页备注输入应为单行文本。
- 交易列表应支持关键词搜索，并可按账户和分类筛选；无匹配结果时应显示明确空状态。
- 交易列表搜索下方的“全部账户/全部分类”筛选项应靠左排列，不应被左右分散。
- 转账交易应同时影响转出和转入账户余额，不应计入收入或支出统计；账户详情应能展示相关转账。
- 从日历点击日期进入交易列表后，应默认筛选当天，同时仍可切换到全部时间等普通筛选。
- 账户详情页应支持修改余额、类型、名称、图标、单行备注、分组、计入资产和隐藏状态；信用卡和储蓄卡支持编辑卡号后四位。
- 添加账户时输入银行或支付平台名称，应自动匹配对应账户图标；信用卡和储蓄卡填写卡号后四位后，资产列表账户名后展示灰色后四位标记。
- 添加账户和账户详情的账户图标选择器应展示通用图标和本地 SVG 图标，列表行左侧为图标、右侧为描述。
- 调整账户余额后，最近交易和首页交易列表应出现对应“余额调整”记录。
- 删除或隐藏账户后，快速记账详情页的账户选择中不应再出现该账户。
- 资产页默认净资产卡片无背景图时，应使用普通卡片背景，不使用蓝色渐变；趋势图应位于资产/负债摘要下方并有足够高度。
- 设置资产背景图后，净资产卡片只能使用轻量黑色可读性遮罩，不应出现蓝色品牌渐变遮罩。
- 资产页没有账户时应展示居中的空状态，提示用户从右上角添加资产。
- 账户分组页没有分组时应展示空状态，提示用户点击右上角加号创建分组。
- 资产页默认展示类型视图；切换到分类视图后，未选择分组的账户应展示在“未分组”卡片中，不能因为 `groupId` 为空而丢失。
- 资产页右上角资产操作菜单应支持在分类视图和类型视图之间切换；切换结果刷新后仍保留。
- 资产页每个账户分区应支持点击标题展开/折叠，折叠状态刷新后仍保留。
- 资产页账户分区展开/折叠应有短动画；开启触感反馈时，展开/折叠也应触发轻触感。
- 资产页分区右侧总额不应统计“不计入资产”的账户。
- 资产页右上角“资产操作”菜单应常驻“排序分组”入口（不再因分区数不足而消失，避免用户找不到）；分区不足 2 个时点击提示“至少有 2 个分组才能排序”，2 个及以上时进入排序模式：所有分区先折叠为标题并显示拖动手柄，可直接拖动手柄稳定调整任意分区顺序（不受展开高度影响，不出现手指与卡片错位、卡顿或第二个分区无法拖动的问题），点击“完成”退出并恢复展开状态，排序结果刷新后仍保留。
- 资产页每个账户分区内应支持长按拖动账户排序，排序按账本、视图和分区分别持久化；开启触感反馈时，分区排序和账户排序应在拖动开始、落点确认和结束时提供轻触感。
- 点击资产卡片右上角图片图标后，应能选择线上图、自定义链接、本地图片和清除背景。
- 本地图片裁剪页应能调节缩放、水平位置和垂直位置，确认后显示裁剪结果。
- 资产背景和头像裁剪页在 Android 真机上不应黑屏，确认后应展示实际裁剪结果。
- Android 端本地图片选择、头像裁剪、JSON 导入和 JSON 导出应在真机上真实可用。
- 头像编辑应复用同一裁剪页，保存后个人页展示新头像。
- 分类管理页新增分类后，应立即出现在记账详情页和全部分类选择器中。
- 分类管理页拖动排序后，记账详情页和全部分类选择器应按新顺序展示。
- 已有交易使用的分类不能删除，应提示使用笔数；未使用分类删除前必须二次确认。
- 数据管理页导出数据后，应下载备份文件（未加密为 zip、加密为 json）；Android 端默认写入系统下载目录，并在成功提示中说明位置。**真机必验**：带图片附件时导出未加密备份 → 初始化 → 导入该备份 → 附件应完整还原（验证 zip 打包/解包在 Android SAF/下载路径上的往返）。
- 设置页点击检查更新后，应打开弹窗查询 GitHub Release；弹窗展示当前版本、最新版本和状态。无更新提示当前已是最新，有更新时展示“下载新版本”按钮，下载期间展示进度，下载完成后打开系统安装确认。
- 设置页底部应展示当前版本；不展示“同步方式”和“Android 打包”说明卡片。
- 快速记账数字键盘默认应触发系统触感反馈，刚安装进入程序时也应生效；关闭设置页“触感反馈”后不再触发。
- 数据管理页导入数据前必须二次确认，确认后应能恢复交易、账户、账本、预算、个人信息和主题偏好。
- 导入空文件或错误格式文件时，应显示失败提示，不应清空当前数据。
- 初始化数据入口必须使用红色危险样式；初始化前必须双重确认，确认后交易、账户、账本、预算、个人信息和主题偏好恢复默认。
- 普通提交和 `main` 推送不应触发 GitHub Actions；只有推送 `vX.Y.Z` 标签后，GitHub CI 才创建 Release 并附加 Android APK。APK 文件名应符合 `verifin-v1.0.0-短提交号.apk` 形式，本机不作为 Android 交付打包环境。
- 连续 GitHub Release APK 应使用同一签名，除切换到稳定签名的首次迁移外，后续版本应可覆盖安装。

## 暂未作为完成标准
- 云同步、登录账号、多端同步和远程备份。
- Android 本机打包；Android APK/AAB 由后续 GitHub CI 生成。
- 标签、项目、商家、退款和高级预算。
- Unsplash API 搜索图库；当前只使用预设线上图片和自定义图片链接。

## 后续优先级
- 继续精修整体视觉系统，重点是资产页、看板页和我的页的信息层级。
- 增加更真实的图标资源和分类图标体系。
- 建立 GitHub CI Android 打包流程。
- 增加更完整的预算周期、预算提醒规则和分类预算汇总能力。
