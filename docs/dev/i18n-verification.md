# 多语言（i18n）真机验证清单

2026-07 完成的多语言迁移的验收指引。安装 CI 构建的 release APK 后按下列顺序验证。

## 实现要点（验证前先了解）

- 语言入口：**我的 → 设置 → 语言**，三个选项：跟随系统 / 简体中文 / English。选择后**即时生效**、无需重启；偏好存本机 KV（`verifin.locale.v1`），不进 JSON 备份，初始化数据时保留。
- 「跟随系统」时按系统语言解析：系统是中文显示中文，其他任何语言显示英文（回落规则：非中文一律英文）。
- **种子数据按首启动语言播种**：全新安装（或初始化数据）时，默认账本名（日常账本 / Daily Ledger）、默认分类（餐饮 / Dining…）、个人简介按当时的语言偏好写入。播种后它们是**用户数据**，切换语言不会跟着变（可手动改名）。
- 有意保留中文、不随语言切换的内容：
  - 隐私政策 / 用户协议**正文**（法律文本以中文为准，标题与按钮已本地化）；
  - 银行 / 品牌图标名（工商银行、支付宝等专有名词）；
  - CSV 导入模板的表头（`日期,类型,金额…`，导入格式兼容所需）与逐行错误提示；
  - WebDAV / 平台桥接（检查更新、SAF 读写）等无界面上下文的**错误消息**——正常流程全部本地化，仅失败提示暂为中文（有意暂缓，未英文化）。

## 一、切换与持久化（5 分钟）

1. 打开应用（中文界面）→ 我的 → 设置 → 语言，确认三个选项显示为「跟随系统 / 简体中文 / English」。
2. 选 **English**：设置页立即变英文（Theme / Language / App lock / Check for updates…），返回后底部导航长按气泡为 Home / Assets / Reports / Me。
3. **杀掉进程重开**：仍是英文，且应用锁（若开启）锁屏文案为英文（Enter password / Biometric unlock）。
4. 切回**简体中文**，确认即时恢复中文。
5. 选**跟随系统**：把手机系统语言切到英文 → 应用显示英文；切回中文 → 应用中文。

## 二、四大 Tab 全页面走查（英文模式，15 分钟）

切到 English 后逐页检查**不应出现任何中文**（上述「有意保留」除外）：

- **首页**：走势卡（Spending trend / Net / Income / Days tracked / Daily average）、预算卡（Jul budget / Remaining / Budget …）、日历卡（Calendar，周表头 Mon–Sun）、最近交易、空态文案；图表点按气泡的日期应为「Jul 5」样式（中文为「7月5日」）。
- **资产**：资产卡（Net worth / Assets / Liabilities）、分区标题（账户类型 Online payment / Cash…）、资产操作菜单、添加账户表单（含校验错误 Account name is required）、账户详情（Balance trend / Statement day / Due day / 还款提醒条）、隐藏账户页、分组管理页、账户报告页。
- **看板**：预算执行 / 分类统计 / 分类明细 / 标签统计 / 日趋势 / 月度收支各面板与空态；右上角统计分析页（时间范围、YoY · MoM 表、分类排行）；面板管理页（开关、排序、恢复默认弹窗）。
- **我的**：宫格入口、账本 / 分类 / 标签管理（含各删除确认弹窗与 SnackBar 提示）、周期记账、个人信息编辑、设置页、数据管理页全链路（导出 / 导入 / CSV / 本地备份 / 加密 / WebDAV / 初始化双重确认 / 检查更新弹窗）。

## 三、记账链路（英文模式，5 分钟）

1. 快速记账数字键盘 → 保存；交易详情页各行（Type / Category / Account / Date / Time / Note / Tag / Mark reimbursable / Refund）。
2. 转账（From account / To account / Fee）、待报销徽标（Reimbursable）、退款冲抵后徽标（Refunded）。
3. 交易列表：筛选（时间 / 排序 / 账户 / 分类 / 标签 / 待报销）、搜索占位、多选批量操作栏（Select all / Category / Account / Delete）、删除确认弹窗。
4. 图片附件行（Attachments / Take photo / Choose from gallery）。

## 四、系统集成点（英文模式，10 分钟）

这些是**无界面上下文**的文案，专门写了按语言偏好解析的通道，需重点验证：

1. **桌面小组件**：语言切到 English 后记一笔（触发推送），小组件标签应变为 "Today's spending"（中文为「今日支出」）。注意：仅在开屏 / 回前台 / 记账后刷新。
2. **记账提醒通知**：English 下设置每日提醒，到点后通知标题 / 内容应为英文（Daily reminder / Don't forget to record today's spending!）。
3. **生物解锁系统弹窗**：English 下触发生物解锁，系统弹窗的标题、取消按钮等应为英文（中文模式下应全中文，无中英混杂）。
4. **首启动播种**：卸载重装（或初始化数据），系统语言为英文时完成引导 → 默认账本应为 "Daily Ledger"、分类为 Dining / Transport…；系统语言中文时应为「日常账本 / 餐饮…」。

## 五、中文回归（5 分钟）

切回简体中文后抽查：首页 / 资产 / 看板 / 我的、一次记账、一次删除确认——确认所有文案与迁移前一致（自动化测试已在中文下覆盖 219 例，真机抽查即可）。

## 已知限制（验证时不算失败）

- 隐私政策 / 用户协议正文为中文（设计决定）。
- 银行 / 品牌图标名为专有名词，不翻译。
- CSV 模板表头与逐行导入错误为中文（导入格式兼容 + 来源均为中文记账软件）。
- WebDAV 连接失败、检查更新失败等**错误提示**暂为中文（有意暂缓，未英文化）。
- 应用语言切换后，已排期的提醒通知按新语言重排要等下一次打开应用或修改提醒设置时生效。
