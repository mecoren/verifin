# Veri Fin UI 规范

## 顶部 Header
所有页面顶部统一使用 `VeriHeader` 或 `PageHeader`。Header 固定高度，支持返回、标题、副标题和右侧操作。不要在页面内手写顶部 `Row + IconButton + Text`，避免不同页面高度和对齐不一致。

## 卡片标题
首页和功能卡片标题统一使用 `titleMedium` 加 `FontWeight.w800`，与“日历”卡片标题保持一致。不要为单个卡片手写更大的标题字号；需要右侧入口时使用统一的标题行组件或同等字号的 `Row`。

## 选择器
项目内的单选项统一使用底部弹出的选择器 `_showOptionSheet`。账户类型、账户图标、账户分组、主题模式、交易筛选和排序都应使用同一套底部选择器。除非是连续输入或系统日期时间选择，否则不要使用 `DropdownButtonFormField` 或页面内分段按钮作为单选入口。

选择器从屏幕底部出现，标题简短，选项使用 44px 以上触控高度，当前选项显示蓝色弱背景和对勾。多级选择应逐级打开新的底部选择器，例如“资产卡片背景”先选择来源，再选择线上图片。

## 弹窗与确认
底部弹窗用于普通选择；`AlertDialog` 只用于破坏性操作、不可恢复操作或必须确认的风险提示，例如删除交易、删除账户、初始化数据。

## 设置与信息行
设置类行统一使用 `SettingsRow`。左侧放图标和标题，右侧数据必须靠右显示；可点击行使用右侧箭头。不要在设置卡片中手写不对齐的 `Row`。

## 卡片交互
卡片默认只用于承载信息。需要点击跳转时应提供明确入口或使用轻量点击反馈；首页趋势卡、预算卡这类大面积入口不使用长按高亮卡片态。

长按只用于低频配置入口，并且必须提供可见提示，例如资产净值卡片右上角的图片图标。图片型卡片使用 `BoxFit.cover` 做封面裁剪，文字层必须增加遮罩或渐变以保证可读性。本地上传的大图应先裁剪压缩后再写入本地存储。

## 图片裁剪
头像、资产背景和后续类似图片入口统一使用独立裁剪页。头像使用 1:1 圆形预览，资产背景使用宽幅封面预览。裁剪页至少提供缩放、水平位置、垂直位置和重置操作，确认后再写入本地数据。

## 金额显示
金额为 0 时统一显示 `0`，使用中性色，不显示 `-0` 或红色。支出、收入和转账颜色只在金额非 0 时表达语义。

## 快速记账弹层
快速记账数字键盘必须是内容高度的底部弹层，不占满全屏。弹层内部保持移动端最大宽度、底部安全区和 4:3 数字按键比例。
