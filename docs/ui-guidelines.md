# Veri Fin UI 规范

## 顶部 Header
所有页面顶部统一使用 `VeriHeader` 或 `PageHeader`。Header 固定高度，支持返回、标题、副标题和右侧操作。不要在页面内手写顶部 `Row + IconButton + Text`，避免不同页面高度和对齐不一致。

## 卡片标题
首页和功能卡片标题统一使用 `titleMedium` 加 `FontWeight.w800`，与“日历”卡片标题保持一致。不要为单个卡片手写更大的标题字号；需要右侧入口时使用统一的标题行组件或同等字号的 `Row`。

## 品牌图标
应用图标源文件位于 `assets/brand/verifin_icon.svg`，当前使用 tessera 风格的浅色圆角底和三层半透明色块。Web favicon、PWA 图标和 Android launcher 图标都应从同一源文件生成，避免不同平台出现不一致的图标风格。

## 选择器
项目内的单选项统一使用底部弹出的选择器 `_showOptionSheet`。账户类型、账户图标、账户分组、主题模式、交易筛选、排序和资产页右上角操作都应使用同一套底部选择器。除非是连续输入或系统日期时间选择，否则不要使用 `DropdownButtonFormField` 或页面内分段按钮作为单选入口。

选择器从屏幕底部出现，标题简短，选项使用 44px 以上触控高度，当前选项显示蓝色弱背景和对勾。动作菜单也使用同一底部弹出样式，但不显示“当前项”对勾，例如资产页右上角的添加账户、管理分组。多级选择应逐级打开新的底部选择器，例如“资产卡片背景”先选择来源，再选择线上图片。

交易列表的搜索和二级筛选应放在同一个紧凑筛选卡片内。关键词搜索使用可清空输入框，账户和分类继续使用底部选择器；筛选后无结果时展示居中的空状态和明确的“换筛选条件”提示。

账户图标选择器使用底部列表，不使用网格 Tag。每行左侧展示实际图标，右侧展示名称和分组；SVG 账户图标统一放在 `assets/account_icons/`，渲染入口使用 `AccountIconBox`，不要在账户列表中直接手写 `Icon(iconForCode(...))`。

记账详情页“全部分类”弹层使用列表行展示分类，不使用多列 Tag，确保分类名称较长时仍可读。多级分类按层级缩进渲染，父分类行尾用展开/收起箭头切换子分类，点选任意层级（父或子）即选中该分类。

系统日期和时间选择器必须走中文本地化，避免出现英文月份、按钮或时间制式文案。

## 弹窗与确认
底部弹窗用于普通选择；`AlertDialog` 只用于破坏性操作、不可恢复操作或必须确认的风险提示，例如删除交易、删除账户、导入数据和初始化数据。

选择账户一律使用账户选择弹窗（`showAccountPickerSheet`），行样式与资产页账户列表一致：左侧账户图标，中间账户名称（含灰色卡号后四位），右侧余额（负数红色、非负青绿色、不计入资产弱化色）。记账页的账户选择框前置图标必须跟随所选账户的图标，不使用固定图标。

## 设置与信息行
设置类行统一使用 `SettingsRow`。左侧放图标和标题，右侧数据必须靠右显示；可点击行使用右侧箭头。设置行保持紧凑但触控高度不低于 44px，不要在设置卡片中手写不对齐的 `Row`。

数据导出使用设置行直接触发 JSON 下载，并用轻量提示反馈结果，提示文案必须说明 Android 端文件位于系统下载目录。Android 端优先通过系统 Downloads 写入备份文件，不依赖文件保存弹窗；Web 端使用浏览器下载。数据导入必须先展示确认弹窗，说明会替换当前本地数据；确认后再打开文件选择器，失败时使用清晰错误提示。检查更新使用设置行触发，不自动检查；点击后打开弹窗展示当前版本、最新版本和状态。有新版本时由用户点击“下载新版本”，下载期间必须展示进度条或百分比，下载完成后再打开系统安装确认。Web 预览环境只展示不可用提示。

账户详情中的类型、名称、图标、单行备注、分组等字段应直接做成可点击设置行；信用卡和储蓄卡需要显示并可编辑卡号后四位。计入资产和隐藏账户使用紧凑开关行，不使用完整高度的 `SwitchListTile`。

隐藏账户不混入普通资产分组；资产页分组列表末尾显示“n个隐藏账户”入口，进入独立页面查看隐藏账户列表。

资产页账户列表支持分类视图和类型视图。右上角资产操作菜单负责切换视图；每个分区标题行必须可点击展开/折叠，右侧展示分区总额、分区拖动柄和展开箭头。分区展开/折叠使用短动画和箭头旋转，开启触感反馈时应同步触发轻触感。分区整体支持拖动排序，拖动手柄按下后必须先进入临时折叠态，再由系统长按拖拽接管；拖动时所有分区和被拖动浮层都临时折叠，松手后恢复用户原来的展开/折叠状态；分区排序按账本和视图模式持久化。分区内账户只通过长按拖动排序，排序状态按账本、视图模式和分区分别持久化；拖动开始、排序落点和拖动结束在开启触感反馈时应提供轻触感。新增账户默认进入所属分区末尾的未排序区域，不主动重排用户已有顺序。信用卡和储蓄卡如填写卡号后四位，账户名称后展示灰色 `(1234)` 标记。

分类管理页使用统一 Header、分段类型切换和紧凑列表行，并按多级树形结构缩进展示：父分类行左侧用展开/收起箭头切换子分类，子分类按深度缩进。Header 的加号新增顶级分类；行内动作表提供重命名、换图标、新增子分类、移动到（选择新父级或“顶级分类”）、删除。排序使用右侧拖动手柄且仅在同级之间生效，不把整行变成拖动区域。删除分类属于破坏性操作，必须先确认；已有交易引用的分类、以及仍有子分类的父分类只能提示原因，不能直接删除。系统分类（余额调整）不显示移动/删除。

## 卡片交互
卡片默认只用于承载信息。需要点击跳转时应提供明确入口或使用轻量点击反馈；首页趋势卡、预算卡这类大面积入口不使用长按高亮卡片态。

长按只用于低频配置入口，并且必须提供可见提示。资产净值卡片背景改为点击右上角图片图标进入设置，不使用长按。图片型卡片使用 `BoxFit.cover` 做封面裁剪，文字层必须增加遮罩或渐变以保证可读性。本地上传的大图应先裁剪压缩后再写入本地存储。

资产净值卡片默认无背景图时使用普通卡片背景和普通文本色；只有用户设置背景图后才使用轻量黑色可读性遮罩和浅色文字，不叠加蓝色、紫色等品牌色渐变。卡片内容顺序为：净资产标题和背景图标、净资产金额、资产/负债摘要、底部资产趋势图。资产趋势图高度需要明显高于普通小图，避免显得压扁。

## 图片裁剪
头像、资产背景和后续类似图片入口统一使用独立裁剪页。头像使用 1:1 圆形预览，资产背景使用与实际净资产卡片一致的宽幅封面比例预览。裁剪页至少提供缩放、水平位置、垂直位置和重置操作，确认后再写入本地数据。本地图片以 data URL 保存时，预览和正式展示必须使用内存图片渲染，不要用网络图片组件直接加载 data URL。

## 金额显示
金额为 0 时统一显示 `0`，使用中性色，不显示 `-0` 或红色。支出、收入和转账颜色只在金额非 0 时表达语义。

记账详情页顶部大金额颜色跟随当前类型：支出用 `veriExpense` 红色，收入用 `veriIncome` 青绿色，转账用 `veriBlue` 蓝色。

## 预算页面
预算页必须同时提供状态概览和编辑入口：顶部使用月份切换器，主体展示预算进度圆环、本月支出、剩余额度、剩余日均和预算金额。预算建议使用独立信息卡，不用长段说明。近 6 月趋势使用支出柱和预算线，必须带月份标签和纵轴刻度。历史对比只展示本月/上月支出、支出环比和预算使用率变化，保持为紧凑摘要，不做复杂图表。预算历史页展示最近 12 个月汇总，每行可进入对应月份预算页。分类预算使用紧凑列表行展示分类、已花金额、预算金额、上月同分类支出和细进度条；列表按已超支、临近超支、有预算、仅有支出、未设置排序。首页预算卡只展示最需要关注的单个分类提醒，正常状态不打扰用户。点击分类行进入金额输入，不新增复杂图表。预算金额输入框保留可见标签和帮助文本，保存入口放在统一 Header 右侧。

## 快速记账弹层
快速记账数字键盘必须是内容高度的底部弹层，不占满全屏。弹层内部保持移动端最大宽度、底部安全区和 4:3 数字按键比例。

数字键、退格、`C`、小数点和正负切换使用统一弱背景按钮，不使用蓝色边框；浅色模式为浅底深字，深色模式为深底浅字。`OK` 可点击时使用主蓝色背景和白色文字，不可点击时使用清晰的浅蓝灰禁用态。

数字键盘按键默认触发系统触感反馈；设置页必须提供“触感反馈”开关，关闭后数字键、退格、清空、金额确认都不触发触感。

## 图表交互
所有数据图表都必须支持交互，后续新增图表同样遵守。曲线图和柱状图统一使用 `InteractiveTrendChart` / `InteractiveBarChart`（`lib/app/chart_painters.dart`）：点击或横向滑动选中数据点/柱子，绘制竖向参考线和高亮点，并通过 `drawChartTooltip` 弹出统一样式的数据气泡（深色底、浅色文字，自动夹紧在画布内，浅色/深色/图片背景下都可读）；再次点击同一位置或点击图表区外取消选中。多序列图表（如预算趋势的预算线+支出柱）气泡内用同色小圆点区分序列。看板环形图点击分段后中心展示该分类的名称、金额和占比，其余分段弱化。

图表区域必须拦截点击：位于可跳转卡片内的图表（如首页支出走势卡）点击只展示数据，不触发卡片跳转；跳转仍由卡片其余区域承担。

## 图表周期
首页趋势和看板日趋势默认使用 7 天周期窗口，不直接把整月日期塞进横轴。日期标签要跟实际窗口一致，月末不足 7 天时展示到月末即可。

看板圆环图必须横向居中，中心只保留金额等核心数字；分类名称通过与圆环同色的细线从圆环外侧引出，不在右侧放“最高分类”等说明块。分类统计至少展示支出金额最高的 5 个分类，剩余分类统一合并为“其他”。圆环尺寸保持克制，接近首页预算圆环的视觉量级。月度柱状图必须提供纵轴标签；超过 `10000` 的刻度使用 `1w`、`1.5w` 等短格式。

看板预算执行卡放在看板顶部，展示本月预算、支出、剩余/超出、分类预算数量和超支数量。该卡只做执行摘要，不承载预算编辑；编辑仍从首页预算卡进入预算页。
