# 贡献指南

感谢参与 Veri Fin（包名 `verifin`）——一个免费、数据自主、本地优先的 Flutter 记账应用。本文件是快速上手入口，完整规范见 [`AGENTS.md`](AGENTS.md)，架构说明见 [`CLAUDE.md`](CLAUDE.md)。

## 环境准备

- Flutter stable channel（`flutter --version`），Dart 随之附带。
- Java 17（Android 构建用）。
- 本地预览只在 **Android 模拟器或真机** 上进行；不在本机打 APK 作为交付物（APK 只由 GitHub CI 在打 `vX.Y.Z` 标签时构建）。

```bash
flutter pub get                     # 安装依赖（同时生成 l10n）
flutter run -d <android-device-id>  # 本地预览
flutter analyze                     # 静态检查
flutter test                        # 全部测试
dart format .                       # 提交前格式化
```

## 上手路线

1. 读 [`CLAUDE.md`](CLAUDE.md)：架构总览（单 Controller + InheritedNotifier、KV vs SQLite 分级、平台条件导入两件套）。
2. 读 [`docs/dev/components.md`](docs/dev/components.md)：**写任何组件前先查这份组件清单**，有现成的就复用。
3. 读 [`AGENTS.md`](AGENTS.md) 的「代码规范 · 组件化与工程约定」：组件化、图标入口、弹窗 helper、错误处理与日志、异步守卫、l10n。
4. 关键技术选型见 [`docs/dev/tech-decisions.md`](docs/dev/tech-decisions.md)；已知限制与技术债见 [`docs/dev/known-limitations.md`](docs/dev/known-limitations.md)。

## 提交前检查清单

- [ ] `dart format .` 已格式化（CI 有格式门禁）。
- [ ] `flutter analyze` 无告警。
- [ ] `flutter test` 全绿；改了功能就补 / 改对应测试。
- [ ] 新用户可见文案已写入 `lib/l10n/app_zh.arb` + `app_en.arb` 并经 `AppLocalizations` 引用（不硬编码中文）。
- [ ] 新组件已先查组件清单、优先复用；确有新增则登记进 `docs/dev/components.md`。
- [ ] 涉及本地存储/图片/文件/系统入口的功能，确认 Android 端有真实实现并检查权限、持久化、进程重启行为。
- [ ] 数据结构变更同步了导出/导入、初始化、样例备份 `docs/dev/verifin-sample-backup.json`。
- [ ] 用户可见改动已往 `CHANGELOG.md` 的 `## [Unreleased]` 加一行。
- [ ] 变更影响命令/架构/流程/配置/用户行为时，同步更新了 `README.md` / `AGENTS.md` / `docs/`。

## 提交信息

- 用 `type: summary` 格式：`feat:` / `fix:` / `refactor:` / `docs:` / `chore:` / `test:`。例：`feat: 增加账户分组图标预览`。
- **提交中不要包含任何 AI 署名**（`Generated by ...`、`Co-authored-by: ...` 等）——这是仓库明确规则。
- 每完成一个相对独立的模块就提交一次，不要把所有改动堆到最后一次提交。

## Pull Request

- 说明变更内容、动机、测试结果；关联 issue（如有）；UI 变更附截图或录屏。
- PR 会触发 CI（格式 + analyze + test），需全绿。

## 发版（仅维护者）

- 发版必须经明确同意；打 `vX.Y.Z` 标签会触发 CI 构建并发预发布，不可逆。
- 流程：先把 `CHANGELOG.md` 的 `## [Unreleased]` 改名为版本号加日期、其上新开空 `Unreleased`，提交后跑 `scripts/publish.sh <patch|minor|major>`（Windows 用 `scripts/publish.ps1`）。
- CI 默认发**预发布**（不标 Latest）；真机验收通过后在 GitHub 手动提升为正式版。
